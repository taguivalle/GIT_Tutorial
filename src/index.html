Introducción

Git es un sistema de control de versiones distribuido; GitHub es una plataforma que aloja repositorios Git y facilita la colaboración.

Conceptos básicos

Repositorio
Contenedor que guarda todo el historial de cambios.
Commit
Instantánea del proyecto en un momento dado.
Branch (rama)
Línea paralela de desarrollo.
Merge
Unir cambios de una rama a otra.
Staging area
Zona intermedia donde se preparan los archivos antes del commit.

Instalación

En macOS:

brew install git

En Ubuntu/Debian:

sudo apt-get update && sudo apt-get install git

En Windows: descarga el instalador desde git‑scm.com.

Comandos esenciales

Comando Descripción Ejemplo
git init Crea un nuevo repositorio local. git init mi-proyecto
git clone Clona un repositorio remoto. git clone https://github.com/usuario/repo.git
git status Muestra el estado de los archivos. git status
git add Agrega archivos al staging area. git add .
git commit Registra los cambios. git commit -m "Mensaje"
git push Envía commits al remoto. git push origin main
git pull Trae y fusiona cambios del remoto. git pull origin main
git branch Lista o crea ramas. git branch nueva-funcionalidad
git merge Fusiona una rama en la actual. git merge nueva-funcionalidad

Flujo de trabajo típico

  1. Crear repositorio (`git init`).
  2. Añadir archivos (`git add`).
  3. Hacer commit (`git commit`).
  4. Crear repositorio remoto en GitHub y enlazarlo (`git remote add origin …`).
  5. Push al remoto (`git push -u origin main`).
  6. Crear ramas para nuevas funcionalidades, trabajar, mergear, etc.

Trabajar con GitHub

Pasos básicos para crear un repo remoto:

  1. En GitHub, pulsa New repository.
  2. Selecciona nombre, descripción y visibilidad.
  3. No marques “Initialize with a README” (ya lo tienes).
  4. Copia la URL del repo (HTTPS o SSH).
  5. En tu terminal: git remote add origin <URL>
  6. Empuja tu rama principal: git push -u origin main

También puedes usar Fork, Pull Request y Issues para colaborar.
